sk8rdude461 August 4, 2016, 9:57am #4

Here’s what I did to solve this (My system may be different than yours, but I know I had the same issue).

I added an array of booleans to player.java

public boolean[] playerSkilling = new boolean[24];

And I called them based off their skilling IDs. So if you were using attack, it would be playerSkilling[0] and so on.

Then in Fishing.java I made them set to true/false based off what I was doing:

c.playerSkilling[10] = true;//Just started fishing

And then in the stop method for my fishing (I use cycleEvents)

					public void stop() {
						resetFishing(c);
					}

resetFishing:

	public static void resetFishing(Client c) {
		c.startAnimation(65535);//Non-existent anim, so it will just reset your player to normal.
		c.getPA().removeAllWindows();//Clears any windows that may have been opened somehow
		c.playerSkilling[10] = false;//Set boolean to false
		for(int i = 0; i < 11; i++) {//Flush skill data
			c.playerSkillProp[10][i] = -1;
		}
	}

And then finally walking.java:

		if(c.playerSkilling[10] == true){
			c.playerSkilling[10] = false;
		}

This will My cycle event updates rather fast, and there’s multiple checks to see if it’s true, and it will never start when walking
