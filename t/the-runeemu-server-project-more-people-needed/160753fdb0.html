I have more rep than zstars, so here it goes.

I have noticed that at the moment there are very few, if any, ‚Äúprofessionally‚Äù made runescape servers. Although Runescape is relatively simplistic for a MMORPG, it is still a MMORPG. MMORPG servers are especially complex projects, which involve many areas of Computer Sciences knowledge, including Relational Databases, Distributed Systems, Event-Controlled Systems, Multi-Threaded design, Object Oriented Design, and others.

With that in mind, I have decided to start creating a new, well designed server.

The Main Language

The language will, of course, be C++. C++ is one of the most widely used professionally used programming languages. Most games, if not all, are written in C++. That includes Quake 3, Quake 4, Half Life 2, World of Warcraft, Starwars Galaxies, Warcraft 3‚Ä¶

In fact, the Runescape server is probably done in C++ too. The Runescape client is done in Java because Jagex is a browser-games firm. If you don‚Äôt believe this, have a look at jagex corporal page, at http://www.jagex.com. You will notice that they are hiring programmers who know either Java OR C++.

Database

We can basically state that a serious MMORPG server needs to make use of a modern relational database. The server will use MySQL, as it is free, fast, and powerful.

Scripting language

Having to re-compile the server to add minor features is far from professional. Modern games make use of scripting languages to define their behaviour, so that new features are safer and faster and easier to add (I think Jagex calls theirs RuneScript). Although it is soon to talk about this, we will probably be using Lua or Python as our scripting language.

Server Design

Although true MMORPG servers are completelly distributed (clusters), such a design is quite complex. The first version of RuneEmu will not be fully clustered, though it will be designed in such a way that it can be modified to add full clustering support in the future.

The design of RuneEmu will be multi-layered and as modular and object oriented as possible, so that the different classes and modules are as reusable as possible.

The target

The target is a very stable server, superior, especially for bigger servers, easily customizable through scripts, and efficient, able to handle many players easily thanks to a proper use of a relational database and a modern, professional and careful modular design.

Who we are looking for

At the moment, we need (quite obviously) more programmers. Although someone with C++ experience would be especially nice, I know that being that strict I won‚Äôt find anyone.

For that reason, everyone with some notion of programming is welcome, as long as he is willing to learn C++ and is able to follow the RuneEmu design.

C++ is very similar to Java, so, those who know Java already have advantage.

What you get

Learning is the main reward. Working on a big, complex project, with the help of someone experienced, is one of the best ways to get good at programming and C++. There are some things that you can never learn until you have worked on something like this.

Of course, we are a team, so you will get credits for what you do as well.

We also could use someone with MySQL knowledge.

If you are interested, please contact me. (Scroll down to see how you can).

Requirements for joining

  • Knowing what programming is. (Not much knowledge required).

  • WILLINGNESS TO LEARN, and to become a good programmer.

  • ABILITY TO FOLLOW my advice / RuneEmu design.

Open / Closed source

For now, the server will be closed source. Eventually though, it will probably go Open Source, at least partially.

Work-flow

I will design the server, in a quite detailed way, so that not-so-experienced people don‚Äôt have much trouble finding something they can do, and so that they can learn without worrying too much about messing up the design badly or adding fatal bugs.

Specifics
