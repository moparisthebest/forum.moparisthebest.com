matzie August 4, 2016, 1:03pm #9

1: Download their client
2: un-jar, decompile client.class
3: look at login protocol
4: use a fully decompiled client of the same revision/type - they all have so many mirrors.
5: trace code logic, make sure login protocols match between the two clients
6: make sure UID is correct from signlink.java (decompile and check yours) if not, change yours to theirs.
7: check if it has any type of mac-address verification on the client. You will want to override this.
8: implements your own way of creating data packets - typically use reflection. also add your noclip to your decompiled client.
9: compile and run using your client, not theirs.

That is a basic guide to making your own cheat client. Modify what you have, and possibly add a external frame for packet sending (like i did)
