    String themePath = null;
    int switchInt = 0;
    if (type.equals("Motif"))

      switchInt = 1;
    else if (type.equals("System"))
      switchInt = 2;

    switch (switchInt) {

        break;
      case 1:
        themePath = "javax.swing.plaf.metal.MetalLookAndFeel";

        themePath = UIManager.getSystemLookAndFeelClassName();
        break;
    }

    try {
      setDefaultLookAndFeelDecorated(false);
      UIManager.setLookAndFeel(themePath);
    } catch (Exception e) {
      setDefaultLookAndFeelDecorated(true);
      System.out.println("[ERROR] Using default theme!");
    }
  }

I assume you mean that when the application only changes the look and feel of the jdialog, the jframe is already loaded and displayed? If so, you need to manually refresh the jframe - java doesn‚Äôt automatically do it during runtime. Just add this snippit after you call the UIManager.setLookAndFeel():

SwingUtilities.updateComponentTreeUI(myJFrame);

The problem is that if you change the look and feel, the components‚Äô sizes may get messed up‚Ä¶so if you need to chage some things such as the frame size, you need to do that after calling the updateComponentTreeUI() method and then repack the frame.

Hope that works for you
