I converted the old OCR1.1 (better known as ELMO) source from C++ to java some time ago and I recently realized it was just collecting dust on my computer. If you actually want to implement it into a bot (it can be done quite easily) just take a look at OCR.java and try to make the bitmap converter take pixels that the server gives the client for sleep words. If you use it, I’d like some credit, and whoever made it(I’m not sure who originally made it), so just acknowledge that it’s ELMO ported to java. It’s not quite as accurate as leosleep aka FOCR, although it’s useful in the sense that if you implemented it properly you wouldn’t need to set anything up. The naming conventions are pretty bad, because I used the ones in the C++ source, oh well

http://www.uploadyour.info/download.php?filename=ocr12884.7z

I realize probably no one has interest in this here, just posting it incase anyone does
