system July 31, 2016, 3:03am #47

Peter: Serializable does not have anything to do with thread-management at all. What Serializable does, is to make it possible to serialize the class in question. If you serialize an object-instance of a class, you write that objects current state to disk in a form that is deserializable if you use the exact same non-modified class as base.

Anyhow, yes, the Client class implements Runnable, and is therefore runnable in a new Thread. This Client class have an EventQueue, which is a place to put Events to be processed. It also got a reference to an Area (the Area where the Player currently is located), which also got an EventQueue, that is public for everyone inside that particular Area to use. The Client class processes all Events that come in to the different EventQueues (both the public one as the private one). None of the Events are getting removed from the EventQueues, but overwritten when no room is left. Currently (in the released version), it can hold up to 200 Events at once without overwriting. I actually havn‚Äôt had any problems with thread-synchronization what-so-ever, that I have experienced. And hopefully, none will exist either‚Ä¶

Creating a large number of connections in a short period of time impacts RSCDv25 heavily, it causes CPU to shoot through the roof and in most cases run out of resources and crash. It can however be fixed easily using checks and just returning if an IP (or multiple IPs) connects at a set threshold.

system July 31, 2016, 3:03am #51

KO9: Yeah, the processor can only run one thread at a time, but the thread is shifting every 10 millis (or similar), which also is OS-specific. And waiting a certain amount of millis (with for instance Thread.sleep(#)) does not necessarily mean that the thread will be paused for exact that time.

Anyway, what causes the most latency-decreasment in RSCDEF and FreeScape (2) is probably the EventQueue-system, and the Area-system. The EventQueue-system because it can handle many Events in very short time. The Area-system because it does not have as many ground-items, ground-objects, wall-objects, npcs and players to iterate through, as the World-system in RSCD-based servers.
