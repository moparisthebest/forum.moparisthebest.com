Ok, In this tutorial I will teach you how to make a powerminer.

First, we must learn the basics, when you open up scar, you will see this.

At the start, you will see

program New; begin end.

Ok, program New; is the name of your script, so, lets call it, Retards Bumfluff, it would be
Program retardsbumfluff; . It must have no spaces and must have a semicolon on the end (

Begin and end mean, well, the beginning and the end of the script, I usually put that right at the end to use it as the Main Loop. You will see what I mean later.

Ok, so you must put the name, and space it out, as so.

[code]program New;

begin
end.[/code]

INCLUDES

Now, there are these lovely things called Includes, or librarys. They are made by lovely nice people to make our life easier. An include is a compilation of functions, procedures, scripts and whatnot. The one we will be using is called SRL, be sure to download this and install it. To use an include in your script you must type {.include SRL/SRL.scar}

It should now look like this.

[code]program New;
{.include SRL/SRL.scar}

begin
end.[/code]

CONSTANTS

These, are things that cannot change, like a username, a password, a place, a name.

A color of rock, Hint Hint.
To declare a Constant you would put.

Const- meaning constant, then you would put a name

Like.

Const Rockcolor = 21430741;

That means, the color of the rock is, well, color 2134074.

Const waittime = 1337;

This means, the time you wait between hitting a‚Ä¶ rock is 1337 miliseconds, just over a second.

We will use these further on, so, follow my coding -)
We will be using these constants.

[code]program New;
{.include SRL/SRL.scar}

const rockwait= 2000;
const rockcolor=9859920;

begin
end.[/code]

You must put a const before each constant, and finish off with a semicolon, we cant get enough of semicolons can we.

MAKING A PROCEDURE

Procedures are a series of tasks. You group tasks together to acheive a goal, like saayy, mining a rock XD.
To mine a rock, you would make it click the rock, and voila.

To make it click the rock, it would detect a certain color, move the mouse there, and click it.

This is where we will be using constants.

Now is also where we will be learning some functions, here is a list of the functions we will use.

IMPORTANT PREMADE FUNCTIONS OR PROCEDURES:

function FindColorSpiral(var x,y: Integer; color, xs, ys, xe, ye: Integer): Boolean; - find color in box specified by xs, ys, xe, ye but start from x,y.

procedure MoveMouse(x,y: integer); - simulate mouse movement, move cursor to x,y instantly.

Mouse(x,y: integer, tolerance:integer,tolerance:integer,true/false)- Clicks mouse.

Istextat2

Ok, that looks all very confusing im sure, but now i shall explain it properly

FindColorSpiral is a very important one, it is used like this

findcolorspiral(x,y,rockcolor,3,3,515,336)

If you look between the brackets, Rockcolor, is the constant,3 and 3 is tolerance by pixels and 515, 336 is the boundrys of the screen to search.

Mouse is the function that clicks the mouse. It is used like this.

mouse(x,y,2,2,true)

In the brackets, X and Y are the coords that are stored from Findcolorspiral and 2,2 is the tolerance, in pixels again, and true.false is Left or right click.

True = Left click

False= Rightclick

All mouse functions will use this.

Istextat2 is the function that looks for text, in this case, we will be looking for the Text, ‚ÄòMine‚Äô in the corner of the screen.
It should say Mine Rock.

If it does, it will click.

You will put these functions together into a procedure like this.

procedure minero; begin repeat findcolorspiral(x,y,rockcolor,3,3,515,336) mmouse(x,y,3,3) istextat2(9,9,'Mine',20) mouse(x,y,2,2,true) wait(rockwait+random(300)) until(invfull) if (invfull = true) then end;

You will notice i added a few things, you will see Repeat at the start and until(invfull) at the end.
This means to repeat mining until your inv is full.
It says, If Invfull to end the procedure. I also added a wait, this is using your rockwait constant, and i added a random 1-300 miliseconds onto it.

Now, we will be using a PREMADE procedure, which is included from SRL. These are used to make scripting less tedious and hard. It will require less scripting. But if you really want to improve, you will make your own procedures.

The premade procedure we will use is called Dropto, it drops items from your inventory.
Your inventory has 28 items in it, and dropto drops from one spot, and stops at another, so if you say Dropto(5-28), it would drop from slot 5 to slot 28.
We will be using dropto(2-28) leaving the pickaxe in the inventory, but dropping all the ores.

The code will look like this.

Procedure dropal; begin dropto(2-28); end;

This drops from 2 to 28.

Now we shall move on to the Main Loop

MAIN LOOPS

This is the begin, end. at the end of the script. This is where you will call upon all the procedures you have made.

The most important is SetupSRL; This is MANDONTORY, it sets up SRL and the script cannot work without it.
You will then use your procedures minero; and dropal;. This is how you will do it.

begin SetupSRL; repeat minero; dropal; until(false) end.

Simple as that, you dont have to write anything because you‚Äôve already done it. Its the code that runs the script.

The repeat and until(false) make sure that the script runs forever. Until(false) means that it never stops repeating.
We only put it there because me must.

Here is what the script should look like in the end.

[code]program New;
{.include SRL\SRL.scar}

const rockwait= 2000;
const rockcolor=9859920;

procedure minero;
begin
repeat
findcolorspiral(x,y,rockcolor,3,3,515,336)
mmouse(x,y,3,3)
istextat2(9,9,‚ÄòMine‚Äô,20)
mouse(x,y,2,2,true)

until(invfull)
if (invfull = true) then
end;

Procedure dropal;
begin
dropto(2-28);
end;

begin
SetupSRL;
repeat
minero;
dropal;
until(false)
end.[/code]

And remember, its not the end result thats important. Its what you do to make it that is important.
