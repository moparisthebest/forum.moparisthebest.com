For some reason it seems a few people on this website are anti-*buntu, for no good reason as far as I can see, so I figured I would write something up as to the reasons why I choose kubuntu and why it would be a good idea for others to do the same. Here we can debate this and maybe even come to a mutual agreement.

Reasons to choose kubuntu (or ubuntu, xubuntu, etc etc):
[b]

  1. The apt-get package manager. [/b] apt-get is simply superior to all other package managers that I have worked with or heard about. I will review them here.

a. RPMs: This is the package type of mandrake, SuSe, fedora and red hat, along with numerous others. However an RPM made for one distro most likely flat out won‚Äôt work at all for another distro, oh, it will probably install, but just break a lot of things along the way. For more in depth info read here: http://distrowatch.com/dwres.php?resource=article-rpm

b. Portage: Gentoo‚Äôs package manager, it downloads the source code and compiles everything specifically for your machine. Sounds great right? Wrong. Not only does it take potentially hours to install certain packages, you can‚Äôt ever upgrade your entire system without almost certainly breaking it entirely. But you get a large performance gain from compiling everything from source right? No, not really. While with certain things such as kernels you may get a performance gain (but only if you know exactly what you are doing, portage doesn‚Äôt and can‚Äôt detect everything and therefore must aim for the general case anyway) the vast majority of your programs won‚Äôt compile any differently on different systems anyway, so you therefore would be better off or at least just as well off downloading pre-built binaries.

c. Apt-get: The debian package manager, which is also the one used by kubuntu, ubuntu, xubuntu etc. Packages from different distributions are almost invariably compatible with each other (I have never found one that isn‚Äôt), even repositories are compatible. There is a program that comes in handy sometimes called alien that will convert RPM packages to DEB packages. You want to compile your programs like gentoo? Certainly, just install the source package and with one command compile your program into your own DEB package that you can then simply install with apt, or distribute to other people. This also allows for easy removal of the package if you messed something up, and easy upgrading. Do you want to upgrade your entire system? Apt-get can do that seamlessly without breaking dependencies and so forth. I have yet to find a flaw with this system.

(I realize there are other package management systems, pacman for arch, tgz packages for slackware and probably many other obscure ones, but I haven‚Äôt used them and therefore can‚Äôt write a fair review, if you know enough about these please write a review of your own here.)

As you can see, apt-get is the clear winner here, now onto the next point.

2. Ease of setup. You try to explain to people that linux is better than windows, you even show them your system and impress them. And then you tell them to boot the linux cd to the command shell, type a myriad of complex commands to configure and set up their system exactly like they want it? I don‚Äôt think so. People who have used windows all of thier lives have most likely never touched the command line, and they shouldn‚Äôt have to if they don‚Äôt want to in linux either. Kubuntu provides a livecd that they can boot up and try, and if they like it they can use a graphical installer right there on the desktop to set it up. The default install comes with most things a common computer user would require, much more than a fresh windows install anyway. Sure it may come with a few extras that die-hard linux users don‚Äôt want, but they are easily removed either from a fancy GUI or an ‚Äòapt-get remove‚Äô on the command line.

3. Ease of use. From bootup it is a point and click operation to do practically anything you want, which is what windows users expect. Even most setup and advanced configuration operations have a GUI now. And of course for the diehards there is always a terminal emulator, and the actual terminal for whatever you want to do.

4. Support. I see this as perhaps the most important aspect of using a distro with a large user base. Chances are you will NEVER run into an issue that has not been run into by numerous people, and fixed already. 99% of the time you can type ‚Äòname of your question/problem kubuntu‚Äô into google and come up with hundreds of results with help. If you do run into such an issue where you can‚Äôt find the answer on google, there are always usually over 1000 people online at any given time in the #ubuntu, #kubuntu, and #xubuntu IRC channels on irc.freenode.net, many of which are more than happy to walk you through the issue and get it solved.

Arguments I have seen against using kubuntu:

  1. ‚Äú*buntu is for noobs, use debian‚Äù (insert your preferred distro in place of debian) The way I see it, I can install debian, spend literally hours installing and configuring all of the software I want, and still be left with older software in the repos, and a smaller userbase to refer to. Or, I can install kubuntu, remove a few packages, install a few others, do a little configuring and have an identical system in much less time with up to date repos and a large userbase for support. It just doesn‚Äôt make sense to set up things one at a time when I can do it all at once.

  2. add your own here!
