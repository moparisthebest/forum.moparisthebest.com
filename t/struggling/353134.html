Well, you can run this java application which will bind to 43594, and than you can go to http://canyouseeme.org and test.

[code=Java]import java.io.IOException;

public class Bind {
public static java.net.ServerSocket listener = null;
public static int port = 43594;

public static void main(String[] args) {
	try {
		listener = new java.net.ServerSocket(port, 1, null);
		System.out.println("Bound to port: " + port);
		while (true) {
			java.net.Socket s = listener.accept();
			System.out.println("Connection from: " + s.getInetAddress().getHostAddress());
		}
	} catch (IOException e) {
		e.printStackTrace();
	}
}

}
[/code]

And use this batch code

@echo off title Port Bind javac Bind.java java Bind pause

Your environmental variables aren’t set up =
You can set them up - http://www.moparisthebest.com/smf/index.php/topic,342685.0.html (Scroll down a bit)
Or you can use the direct path in the batch file

@echo off title Port Bind "C:\Program Files\Java\jdk1.6.0_YOURVERSON\bin\javac.exe" Bind.java "C:\Program Files\Java\jdk1.6.0_YOURVERSON\bin\java.exe" Bind pause

Putting the direct path in the batch file is faster, but setting the environmental variables is best in the long run because you don’t have to set the path each time you want to run a different batch file.
