Kabraxis:

You cant send packets in RS. I‚Äôve used WPE pro for client side editing, then I tried sending the same packets, and it does not work because the server is like what the f?

It works fine if you intercept in the right place and make sure that you change the packet header to the proper header (and of course the params/arguments to the proper values as well). Keep in mind that the header has been modified by the client. All they use is an array that is populated with various int values. These int values are added to the original packet id. The client and server use the same modifier values so they are in synch. Meaning: the server sends the client packets with the same modification values so the client can subtract off the current index in that array. When the client wants to send something, it adds that value at the current index and increments the index. If you know the value the client used to modify the packet header, you can very easily change that portion of the packet so that the server will accept the incoming packet. That is only to get the server to accept it. That says nothing about how detectable that method is. (It is very unsafe to do that). Really, the only real reason to exploit something like that is for a dupe/item creation method. Outside of that, there is no real use for it in botting terms.

All this in mind, it is not enough to simply know what modifier will be next. You have to look at the code and see exactly what the client does when it sends game data via packets. It does not simply send a solo packet signifying some action. All actions are preceded by some important information. (A small amount of research will yield you this information). You not only have to send the packet but you also have to build the ‚Äúframe‚Äù that the server is expecting on each and every single solitary action you could possibly perform. If you do something out of sequence, you will not have that character very long.
