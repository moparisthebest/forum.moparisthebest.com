TUTORIAL on GUIs
By: iwasinmoparseas101class

What is a GUI? GUI stands for Graphical User Interface that uses a couple packages supplied by Sun:

java.AWT (Abstract Window Toolkit), which is an older package. While most of the package was replaced by
javax.swing, the AWT classes may still be needed while working with Swing and your GUI all together.

GUIs are made using ‚Äúevent-driven programming‚Äù. EDP uses a sort of ‚Äúsignal and response‚Äù system. The signals used
in EDP are known to most as EVENTS. When an EVENT is sent to a LISTENER, it‚Äôs called ‚Äúfiring the event‚Äù. Usual
objects that fire an event are Buttons (the most common anyway). A single component can have as many listeners
as you desire‚Ä¶even no listeners at all (basically taking up space on the GUI).

In order for a LISTENER to actually handle the EVENT, you must IMPLEMENT ActionListener. ActionListener contains
methods that handle the events which is why they are called ‚Äúevent handlers‚Äù.

| | | | | |
| Component | ----> | Event | ----> | Listener |
| | | | | |

JFrame is what you might think of as the Window. Your class will EXTEND JFrame that way it has a border and basic
buttons to minimize and close the window.

A sample GUI:
--------------------------------------------------------------------------------------------------------------------
import javax.swing.JFrame;
import javax.swing.JButton; //you could do import javax.swing.*;

public class GUIDemo implements ActionListener{
     public static final int WIDTH = 300;
     public static final int HEIGHT = 200;
     
     public static void main(String[] args){
         JFrame window = new JFrame(); //creates basic border and buttons mentioned above
         window.setSize(300,200); //a method that of course sets the dimensions of the window. setSize(width,height)
         window.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);

         JButton end = new JButton("Click to terminate program.");
         end.addActionListener(this);  //adds a listener to see whether or not end (this) is clicked
         
         window.add(end); //adds the button to the GUI

         window.setVisible(true); //makes the GUI visible.  Put false to hide the window
     }

     public void actionPerformed(ActionEvent e){
         String cmd = e.getActionCommand(); //gets whatever component was clicked
         
         if(cmd.equals("Click to terminate program.")){  //notice the String is the button's label
              System.exit(0);
         }
     }

}
--------------------------------------------------------------------------------------------------------------------

The JButton Class:

When creating a new button to add to your GUI, you give a String arguement to the constructor of the JButton and the
String will be displayed on the button. Adding a Button to the JFrame is done by using the .add(Component c) method
However, you can add the Buttons to Containers as well (called JPanels which will be discussed later). With Buttons,
you know that little x in the upper right hand corner to close the window? That doesn‚Äôt belong to JButtons, that‚Äôs a
JFrame object.

Action Listeners and Action Events:

For each button, you need to specify the objects (in this case buttons), that will be listened to. Next, you need
to define the methods to invoke once the event is sent to the listener. For this, we use the ActionListener
interface which requires only one implemented method:

public void actionPerformed(ActionEvent e)

in the above example code, when the JButton end was clicked, the program looked at the actionPerformed method and
said ‚Äúhey‚Ä¶this event happened, let‚Äôs exit the program now.‚Äù So the program terminated because we wrote it to do
so in the if statement. NOTE: Swing programs NEVER end until a System.exit(int arg) is executed.

The JLabel Class:

Labels are basically just lines of text and are created as follows:

JLabel welcome = new JLabel(‚ÄúWelcome!‚Äù);

Labels can be added to a JFrame the exact same as a button using the add(Component c) method. (ex: add(welcome))
Labels can be changed by using the labelName.setText(String newText) method. (ex: welcome.setText(‚ÄúGoodbye‚Äù)

The Color Class:
The Color class is extremely easy to use and all you really need to know are the setBackground(Color c) and
setForeground(Color c) methods. The Color Class comes with the following constants: Black, Blue, Cyan, Grey, Green
Magenta, Orange, Pink, Red, White , and yellow. However, you can create your own color by simply doing:

Color colorName = new Color(int r, int g, int b); the values for r,g, and b can be 0 <= x <= 255

To include alpha in that method, it‚Äôs simply: new Color(r, g, b, a);

BorderLayout frame = new BorderLayout(); //creates the figure shown above

in order to add Components to this layout type, you must specify WHERE to place the component:

For example, let‚Äôs use the Welcome label used before. If we wanted to add it to the NORTH Panel, we‚Äôd simply say:

add(welcome, BorderLayout.NORTH);

There are also FlowLayouts and GridLayouts that are declared the same way as the BorderLayout:

FlowLayout flow = new FlowLayout();

Panels:

A Panel is a object of the JPanel class that groups objects together. The purpose of Panels are this:

EXAMPLE: Let‚Äôs say you are using a BorderLayout() and you want 2 objects in the SOUTH panel. Well, if you did the
add(Component c, BorderLayout.SOUTH);

followed by another:
add(Component b, BorderLayout.SOUTH);

The SOUTH panel would only have Component b because it would overwrite Component c. So how do we fix this? Simple.
We use a JPanel.

--------------------------------------------------------------------------------------------------------------------
setLayout(new BorderLayout());

JPanel labelPanel = new JPanel(new FlowLayout()); //puts objects right after another

JButton one = new JButton("1");
labelPanel.add(one);
JButton two = new JButton("2");
labelPanel.add(two);

add(labelPanel, BorderLayout.SOUTH);
--------------------------------------------------------------------------------------------------------------------

if we did not make the JPanel, only Button 2 would be visible.

Menus:

To add menus we use 3 of the Swing classes. They are:

JMenuItem
JMenu
JMenuBar

The JMenuBar is the actual container for the menus. So if you have multiple menus like File, Edit, Format you‚Äôd
simply create the menus:

JMenu file = new JMenu("File"); //creates the File menu
JMenu edit = new JMenu("Edit"); //creates the Edit menu
JMenu form = new JMenu("Format"); //creates the Format menu

then we‚Äôd need to add MenuItems to each menu:

JMenuItem exit = new JMenuItem("Exit");
exit.addActionListner(this); //listens to see if exit is clicked.
file.add(exit); //adds exit to the File menu.

JMenuItem copy = new JMenuItem("Copy");
copy.addActionListener(this);
edit.add(copy); //adds copy option to the Edit menu.

JMenuItem font = new JMenuItem("Font");
font.addActionListener(this);
form.add(font); //adds the Font option to the Format menu

Now that we have each of the menus created, how will we put them up there? For this task, we‚Äôd use a JMenuBar to
hold all of the menus in.

JMenuBar bar = new JMenuBar(); //notice no argument constructor
bar.add(file);
bar.add(edit);
bar.add(form);

So now our container has all of the menus with each having their own items. How do we put it in our GUI?!

well, JFrame has a method that does it for us!

setJMenuBar(Bar b);

so our code would look like this:

setJMenuBar(bar);

With the Menu class, we can actually have menus within menus just by adding new JMenu‚Äôs to an already made JMenu.

Listeners Used as Inner Classes:

For those of you who know about Inner Classes, bare with me as I explain it to those not familiar with them.

Basically an Inner Class is a PRIVATE (important) class that can use the same instance variables of the class for
which they are contained in. It also saves time from going back and forth from class to class in your JDK.

So basically we‚Äôll have something that looks like this:

--------------------------------------------------------------------------------------------------------------------
public class GUI{

   <some code to create stuff>
       .
       .
       .
   
   private class innerListener implements ActionListener{
         public void actionPerformed(ActionEvent e){
              someObject.setBackground(Color.BLUE);
         }
   }

   <then we have to add the class into our outter class>
   JMenuItem pickChoice = new JMenuItem("Background Color to Blue");
   pickChoice.addActionListener(new innerListener());  //use the inner class as an arguement to the actionListener
}

Text Fields and Text Areas:

Text fields and areas are an obejects of classes called JTextField and JTextArea. These are added to your GUI the
same way as buttons by using the add(Component c) method. To create a Text Field we use this bit of code:

private JTextField name = new JTextField(int n);

n is the number of characters across you want it to be. Usually programmers just set n=10. So they‚Äôd have:

private JTextField name = new JTextField(10);

in order to retrieve text from the JTextField, we use the getText() method. So:

String input = name.getText();

if name was John, then input would also be John.
We can also set the text of the JTextField by using the setText(String s) method:

name.setText(‚ÄúMy name is John‚Äù); //so the name field would be: My name is John

JTextAreas are basically the same as TextFields, HOWEVER, we instantiate it a bit differently:

JTextArea area = new JTextArea(int l, int c);

l is the number of lines you want and c is the number of characters per line. So say we wanted 5 lines and 20
characters per line. How would we do that?

JTextArea area = new JTextArea(5,20);

The JTextArea has a couple of helpful methods like: setLineWrap(Boolean b) where if b is True then characters are
put onto the next line if they go past 20 characters. There is also the setEditable(Boolean b) where if b is False,
then only the GUI program can change the text.
