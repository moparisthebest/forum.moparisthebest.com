I get the feeling from this thread that many of you have absolutely no idea what a memory leak actually is. Just because a program uses memory, or because it uses more memory than you think it should, does not mean it has a memory leak.

PID USER PR NI VIRT RES SHR S %CPU %MEM TIME+ COMMAND 2412 mopar 20 0 637m 207m 34m S 1 2.6 0:48.76 firefox-bin
(I believe that would show up on windows as 207m)
