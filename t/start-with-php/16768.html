[size=10pt]POST[/size]

Edit: POST Test. Link

POST is like GET except when you use POST the variables and data don‚Äôt get sent in the URL. Instead they are stored in your cache, and then extracted and sent to the next page. So if you went to index.php in this example, typed in something in the box, you would get sent to index2.php and whatever you typed in would be shown and it would still be index2.php at the address bar.

index.php should have‚Ä¶

<html>
<head>
<title>POST Example -&gt; index.php</title>
</head>
<body>
<form method="post" action="index2.php">
  <!-- Look at this Input tag. It has a name of msg. This will be our POST name. -->
  Message: <input type="text" name="msg" /> <br />
  <input type="submit" value="Submit" />
</form>
</body>
</html>

index2.php should have‚Ä¶

<?
 // Sets the variable $message equal to the POST variable of 'msg'.
 // 'msg' is the name of our Input Box in index.php.
 $message = $_POST['msg'];
 
 // Check to see if the variable $message equals nothing.
 // If it does equal nothing, then send the user back to index.php.
 If ($message == "") {
   header('Location: index.php');
 }
?>
<html>
<head>
<title>POST Example -&gt; index2.php</title>
</head>
<body>
 <!-- Sends whatever is in the variable $message. -->
 Message: <?=$message;?>
</body>
</html>

POST is generally used for usernames/passwords/user options/ and stuff like that.
